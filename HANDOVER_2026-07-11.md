## Project purpose (one paragraph)

Given a PubMed query, discover papers → fetch PDF + supplementary → find INSDC read accessions → apply a domain-expert **pre-filter** to keep only papers matching a strict scope → map runs to case/control labels → and only THEN download raw fastqs. The current corpus is a 1128-hit CRC-microbiome PubMed query; 510 rows were probed for coverage. Post pre-filter we know 46 of them are in-scope, 37 partial.

## Subagent patterns that worked

- **Personality doc + batch-input TSV** = clean way to fan out a task across N parallel subagents. Wrote each a slightly different prompt referencing the SAME personality doc and DIFFERENT batch file. All wrote to disjoint output paths — no race conditions.
- **6-way batching for a ~350-item corpus**: each subagent processes ~60 items sequentially. That takes 20-40 min per subagent. Any bigger and they hit context pressure.
- **Text files, not PDFs**: subagents that Read PDF files hit the 32 MB API request cap fast. Extract to `.txt` first (`pdftotext -layout` in the pptxplots env → 4-46× size reduction).
- **Europe PMC full-text XML** at `webservices/rest/{PMC}/fullTextXML` is a strong fallback when poppler fails. Recovered 26/26 broken extractions.
- **Recovery pass**: 1 dedicated subagent for the UNCERTAINs (or any bucket that needs re-review) is much cheaper than re-running the whole fanout.
- **Sibling coordination**: give each parallel agent a disjoint set of output files. Include an explicit "your siblings are running now, don't touch their files" line in the brief.

## Subagent patterns that DIDN'T work

- **One subagent processing 19 PDFs in one context**: hit the 32 MB API cap. Text files fixed it, but batching smaller helps too.
- **Trying to write a single Python classifier that calls an LLM API**: no API keys were set in the env, no SDKs installed in `pyhmmer`, and the pyhmmer rule (`don't pip install`) applies. Subagents doing the classification directly is the working pattern.
- **Assuming a naive fetch would work on Cloudflare-gated sites from cluster**: it fails 100% for Elsevier/Wiley/T&F/Cell.com. Requalifying the bulk-fetch list to "likely-OK" (excluding Cloudflare-gated prefixes) was essential.
